# 标准化销售培训Workflow

## 简介
本项目是一个面向企业销售培训场景的 Workflow 驱动智能训练系统。系统采用 Workflow 负责整体训练流程控制，大语言模型负责场景生成与能力评估，并预留 Agent 与 RAG 能力扩展接口，构建兼具标准化与智能化的销售训练平台。

## 技术栈
Python、Workflow Architecture、LangChain、LLM API、LLM-as-a-Judge、Chroma（RAG）

## Workflow 核心架构
1. **场景构建模块（Scenario Generator）**
负责构建完整的销售训练场景。系统通过随机组合产品信息、客户角色以及客户顾虑画像，形成每轮训练的初始业务上下文，并为整个 Workflow 提供统一的训练目标。该模块保证训练场景具有足够的随机性，同时保持业务逻辑的一致性，为后续对话生成和能力评估提供标准化输入。
2. **Workflow 流程调度引擎（Workflow Orchestration Engine）**
作为整个系统的核心控制模块，负责维护训练状态、管理顾虑画像、调度训练流程以及控制训练轮次。系统根据当前尚未解决的客户顾虑动态决定下一轮训练目标，并持续更新训练状态，直至所有关键顾虑得到有效回应或达到预设训练结束条件，从而保证整个训练过程具有可解释、可追踪和可复现的特点。
3. **大语言模型推理模块（LLM Reasoning Engine）**
负责完成训练过程中所有需要自然语言理解与生成的任务，包括基于当前训练状态动态生成客户提问、分析学员回答内容以及识别回答所覆盖的客户顾虑。系统通过 Prompt Engineering 将业务规则与训练目标约束到大语言模型的推理过程中，使生成内容既具有真实性，又能够严格遵循 Workflow 的整体控制逻辑。
4. **智能评估与学习反馈模块（Evaluation & Coaching Engine）**
训练结束后，系统首先利用 LLM Judge 对学员回答进行结构化能力评估，并结合 Workflow 全流程记录生成能力画像。在此基础上，通过企业知识库和 RAG 检索生成针对性的复盘内容、最佳实践以及学习建议，为后续训练提供持续反馈和个性化学习支持。

## 后续开发计划（Future Work）

未来版本将逐步形成 **Workflow + Agent + RAG** 的整体架构。其中 Workflow 负责训练流程控制，Agent 提供自主决策与动态规划能力，RAG 为大语言模型提供企业知识支撑，共同构建更加智能、灵活且可持续演进的企业销售培训系统。

###  Agent 增强训练流程（Agent-Augmented Workflow）
当前版本采用 Workflow 作为整体训练框架，以保证训练流程的标准化、可解释性与稳定性。后续计划在 Workflow 的基础上引入 Agent 能力，使系统能够根据学员历史表现、训练状态以及业务目标，自主调整训练策略、规划训练路径，并动态生成更加个性化的训练场景，在保持流程可控的同时进一步提升系统的智能性和灵活性。

###  RAG 增强知识推理（RAG-Enhanced Knowledge Reasoning）
当前版本的大语言模型主要依赖 Prompt 完成问题生成与回答评估。后续计划引入企业知识库与 RAG 检索机制，将产品资料、销售案例、标准话术、行业规范等知识实时提供给 LLM，增强模型在问题生成、能力评估、训练复盘以及 AI Coach 学习建议等环节的专业性与可解释性，并支持企业级知识持续更新与扩展。
